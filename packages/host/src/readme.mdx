# squid·ware UI

This Storybook is the multi-framework component catalog for the squid·ware ecosystem — React, Vue, and Angular. You copy a component into your app and change it when you need to. No heavy library lock-in.

**Package note:** workspace packages are still `@quido-ui/*` (`q-*` / `qui-*` selectors). Squid·ware product apps import `@squid-ware/ui` for the platform runtime kit; that is a separate package. This catalog is visually branded for squid·ware and published at [ui.squid-ware.nl](https://ui.squid-ware.nl).

This host Storybook is the entry point: shared docs live here, and each framework’s components are linked under Angular, React, and Vue.

## Philosophy

- **Opt-out**: Components work as-is. You can paste the source into your project and edit it.
- **Consistency**: Shared look and feel via CSS variables (design tokens).
- **Accessibility**: Keyboard use, screen readers, and contrast are part of the default.

## Prerequisites

- Node.js **20.19+** (Storybook 10). With nvm: `nvm use` from the repo root (see `.nvmrc`).
- [pnpm](https://pnpm.io/) for the workspace.

```sh
pnpm install
```

## Open Storybook

From the monorepo root:

```sh
pnpm dev           # all Storybooks in parallel
# or individually:
pnpm host dev      # this shell + docs (port 6009)
pnpm vue dev       # Vue components (port 6006)
pnpm react dev     # React components (port 6007)
pnpm angular dev   # Angular components (port 6008)
```

Build everything for deploy (frameworks first, then host):

```sh
pnpm build
```

## Use a component

Packages are private workspace packages (`@quido-ui/react`, `@quido-ui/vue`, `@quido-ui/angular`). Typical flow:

1. Browse the matching Storybook (React, Vue, or Angular).
2. Copy the component file(s) into your app.
3. Include the design tokens CSS for that package (see [Design System](?path=/docs/design-system--docs)).

### React

Source lives under `packages/react/src/components/`. Styles sit in the same `.tsx` file (styled-components + CVA). Tokens: `packages/react/src/index.css`.

```tsx
import { Button } from "./button";

<Button variant="primary">Click me</Button>;
```

### Vue

Source lives under `packages/vue/src/components/` as SFCs with `<style scoped>`. Tokens: `packages/vue/src/assets/base.css`.

```vue
<script setup>
import Button from "./button.vue";
</script>

<template>
  <Button variant="primary">Click me</Button>
</template>
```

### Angular

Source lives under `packages/angular/src/components/`. Prefer directive-style hosts (for example `button[qui-button]`). Tokens: `packages/angular/src/styles.css`.

```html
<button qui-button>Click me</button>
```
